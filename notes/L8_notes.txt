Recap of SQLAlchemy Relationships

- One-to-many: One record in a table can relate to many records in another table (e.g., one customer can have many service tickets).
- Many-to-many: Many records in one table relate to many records in another table (e.g., mechanics can work on multiple tickets, and tickets can have multiple mechanics).

What is Pagination:

- Divides large sets of data into manageable chunks or pages. Prevents APIs from overwhelming users with too much data at once
- Enhances performance and user experience by delivering smaller results, more digestable portions, and reduces server load

Pagination Methods: Limit and Offset: Most common methods for pagination

- Limit: specifies how many results to return (page size)
- Offset or Page: Skips certain number of results (page number)
    Example: (https://api.example.com/products?limit=10&offset=20)
        - returns 10 products, starting from the 21st product (offset 20)

Other technique: Page and Page size
    Example: (https://api.example.com/products?page=2&page_size=10)
        -   Similar to limit and offset but uses page and page_size.
        -   This fetches the second page with 10 results per page.
